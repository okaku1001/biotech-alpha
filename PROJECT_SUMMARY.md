## 💡 技术决策说明

### 为什么选择 Next.js 而不是 Vite？
- 需要 SEO（投资者会搜索公司名）
- 需要 API Routes（隐藏 API Key）
- 需要服务端渲染（首屏速度）

### 为什么选择 Tailwind 而不是 CSS-in-JS？
- 零运行时开销
- 更好的 SSR 支持
- 更小的 bundle 大小
- 更快的开发速度

### 为什么选择 Recharts 而不是 Chart.js？
- 基于 React 组件（更好的集成）
- 高度可定制
- 响应式设计
- 声明式 API

### 为什么选择 Framer Motion？
- 最流行的 React 动画库
- 声明式 API
- 优秀的性能
- 支持手势和拖拽
